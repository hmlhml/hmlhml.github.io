webpack入门-webpack-dev-server

发表于 2018-08-31

一、webpack-dev-server使用方法

作用
  • 实现自动打包编译的功能
  • 页面自动刷新
安装
  • npm install webpack-dev-server -D 把这个工具安装到项目的本地开发依赖
  • 注意：webpack-dev-server要正常运行，必须在本地项目中安装 webpack (webpack4必须安装webpack-cli)
用法

  • 安装完毕后 这个工具的用法 和 webpack命令的用法 完全一样

  • 由于是在项目中安装的，所以 不能把他当做脚本命令，在powershell 终端中直接运行（只有全局安装的工具才能在终端中正常执行）

  • 在package.json中配置启动命令

  • 在终端npm run dev

  • 同时将index.html中引入js路径修改

注意
  • webpack-dev-server 帮我们打包生成的bundle.js文件，并没有存放到实际的物理磁盘上；而是，直接托管到了 电脑的内存中，所以，我们在项目根目录中 根本找不到打包好的 bundle.js
  • 我们可以认为 webpack-dev-server 把打包好的文件 以一种虚拟的形式 托管到了 项目的根目录 虽然我们看不见，但是可以认为 和dist、src平级，有一个看不见的文件，叫bundle.js

二、webpack-dev-server 参数

在package.json中"dev": "webpack-dev-server --open --port 3000 --contentBase src --hot"

  • –open 自动打开浏览器

  • –port 3000 设置启动时候的运行端口

  • –contentBase src 指定托管的根目录

  • –hot 每次修改之后，不会重新打包bundle.js，只是把修改的地方打包更新
